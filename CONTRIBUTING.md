# Contributing

このリポジトリへの改善提案や修正は歓迎します。

## 基本方針
- 組織導入で実際に使える内容を優先してください。
- ガバナンス、運用、教育の観点をバランスよく扱ってください。
- 大きな構成変更は `Issue` で背景共有してから進めてください。

## 推奨フロー
1. `Issue` を作成する
2. branch を切る
3. `docs/` と必要な図表を更新する
4. `./.github/skills-config/ebook-build/invoke-build.ps1` を実行する
5. `VALIDATION_CHECKLIST.md` を確認する
6. `Pull Request` を作成する

## レビュー観点
- 6視点や導入ステップが一貫しているか
- 図表や文章にリンク切れがないか
- セキュリティ・ガバナンス観点が十分か
- 現場で再利用しやすいテンプレートになっているか
